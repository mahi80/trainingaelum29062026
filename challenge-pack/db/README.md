# `db/` — auto-loan warehouse schema + seed

A normalized auto-loan / banking warehouse: **108 tables / 589 columns** across
four schemas, sized and shaped so natural-language questions require real
multi-table joins (the core of the NL-to-SQL part).

## Schemas
| Schema | Purpose | Examples |
|--------|---------|----------|
| `ref`  | reference / lookup dimensions | `credit_band`, `loan_product`, `decision_reason`, `vehicle_make/model` |
| `loan` | operational origination + servicing | `loan_application`, `borrower`, `vehicle`, `underwriting_decision`, `loan`, `payment`, `policy_rule` |
| `doc`  | documents / pages / extractions | `document`, `document_page`, `extraction_cell` (links to `example/` scans) |
| `app`  | auth + sessions + embeddings | `user_account`, `session`, `document_chunk` (pgvector), `schema_embedding` |

## DDL apply order (numbered; FK-safe)
```
00_extensions.sql  vector / uuid-ossp / pg_trgm + schemas
10_ref.sql         16 reference tables
20_core.sql        29 operational tables (party→borrower→application→decision→loan…)
25_doc.sql         documents / pages / extractions / audit
30_app.sql         roles / users / sessions (sessions in Postgres)
50_history.sql     SCD2 *_history (generated by schemagen)
60_xref.sql        many-to-many bridge tables (generated)
70_servicing.sql   amortization / payments / repossession … (generated)
75_extra.sql       fraud / KYC / dealer / telematics … (generated)
80_vector.sql      pgvector: document_chunk, row_embedding, schema_embedding (HNSW)
90_views.sql       analytics views (vw_application_360, …)
```
`docker compose up` mounts `db/ddl/*.sql` into `/docker-entrypoint-initdb.d`, so
they apply in order on first boot, then `seed.py` runs.

## Regenerate the long tail
```bash
python ../tools/schemagen/expand.py --ddl ddl   # rewrites 50/60/70/75/90, asserts >=100 tables / >=200 cols
```

## Seed
```bash
pip install -r seed/requirements.txt
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/postgres python seed/seed.py --seed 1337 --example ../example
python seed/seed.py --dry-run --example ../example     # validate plan without a DB
```
Deterministic (`Faker.seed`). Produces ~5k+ referentially-consistent rows and
**maps all 60 documents / 100 pages from `example/manifest.jsonl`** to
`doc.document` / `doc.document_page` rows pointing at the scans — so structured
records and scanned documents describe the same loans. Default seed users
(password `ChangeMe123!`, argon2id): `admin`, `uw1`, `officer1`, `auditor1`.

Snapshot a fixed dump: `pg_dump --no-owner $DATABASE_URL | gzip > seed/seed_fixed.sql.gz`.

> SSNs are tokenized; all PII is synthetic.
